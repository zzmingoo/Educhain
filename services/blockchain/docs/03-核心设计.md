# 核心设计文档

## 1. 区块链设计

### 1.1 设计思想

本模块实现的是一个**简化的区块链**，专注于数据存证而非去中心化共识。设计遵循以下原则：

- **单链结构**：线性的区块链，每个区块指向前一个区块
- **即时确认**：交易添加后立即打包，无需等待
- **中心化管理**：单节点运行，无需 P2P 网络
- **哈希链保护**：通过哈希链确保数据不可篡改

### 1.2 与传统区块链的对比

| 特性 | 比特币 | 以太坊 | 本模块 |
|------|--------|--------|--------|
| 共识机制 | PoW | PoS | 无（中心化） |
| 出块时间 | ~10分钟 | ~15秒 | 即时 |
| 智能合约 | 无 | 有 | 无 |
| 虚拟货币 | BTC | ETH | 无 |
| 网络类型 | 公链 | 公链 | 私链 |
| 应用场景 | 数字货币 | DApp平台 | 数据存证 |

### 1.3 核心优势

- **合规性**：不涉及虚拟货币，符合监管要求
- **高性能**：无需挖矿和共识，响应速度快
- **低成本**：单节点运行，资源消耗少
- **易维护**：架构简单，便于开发和维护

## 2. 数据结构设计

### 2.1 区块结构（Block）

```python
@dataclass
class Block:
    index: int                    # 区块索引（从0开始）
    timestamp: str                # 时间戳（ISO 8601格式）
    transactions: List[Transaction]  # 交易列表
    previous_hash: str            # 前一区块哈希（64位十六进制）
    hash: Optional[str]           # 当前区块哈希（64位十六进制）
```

#### 字段说明

- **index**：区块在链中的位置，创世区块为 0
- **timestamp**：区块创建时间，格式如 `2024-01-01T12:00:00.000000`
- **transactions**：包含的所有交易，至少包含 1 个交易
- **previous_hash**：前一区块的哈希值，形成链式结构
- **hash**：当前区块的哈希值，由区块内容计算得出

#### 哈希计算

```python
def calculate_hash(self) -> str:
    block_string = json.dumps({
        'index': self.index,
        'timestamp': self.timestamp,
        'transactions': [tx.to_dict() for tx in self.transactions],
        'previous_hash': self.previous_hash
    }, sort_keys=True, ensure_ascii=False)
    
    return hashlib.sha256(block_string.encode('utf-8')).hexdigest()
```

**关键点**：
- 使用 SHA-256 算法
- JSON 序列化时 `sort_keys=True` 确保顺序一致
- 不包含 `hash` 字段本身（避免循环依赖）

#### 证书编号生成

```python
def generate_certificate_id(self, knowledge_id: int, block_index: int) -> str:
    year = datetime.now().year
    unique_str = f"{knowledge_id}-{block_index}-{uuid.uuid4().hex[:8]}"
    hash_obj = hashlib.sha256(unique_str.encode())
    unique_number = int(hash_obj.hexdigest()[:6], 16) % 1000000
    
    return f"CERT-{year}-{unique_number:06d}"
```

**格式**：`CERT-2024-123456`
- `CERT`：固定前缀
- `2024`：年份
- `123456`：6位唯一编号

## 3. 核心算法

### 3.1 内容哈希算法

使用 SHA-256 算法对内容生成唯一指纹：

```python
import hashlib

def calculate_content_hash(content: str) -> str:
    return hashlib.sha256(content.encode('utf-8')).hexdigest()
```

**特点**：
- 固定长度：64位十六进制字符串（256位二进制）
- 唯一性：内容不同，哈希值必然不同
- 不可逆：无法从哈希值反推原内容
- 雪崩效应：内容微小变化导致哈希值完全不同

### 3.2 区块链验证算法

```python
def is_chain_valid(self) -> bool:
    for i in range(1, len(self.chain)):
        current_block = self.chain[i]
        previous_block = self.chain[i - 1]
        
        # 验证当前区块哈希
        if current_block.hash != current_block.calculate_hash():
            return False
        
        # 验证与前一个区块的连接
        if current_block.previous_hash != previous_block.hash:
            return False
    
    return True
```

**验证步骤**：
1. 遍历所有区块（从第1个开始，跳过创世区块）
2. 重新计算当前区块哈希，与存储的哈希比对
3. 检查当前区块的 `previous_hash` 是否等于前一区块的 `hash`
4. 任何一步失败，整条链无效

### 3.3 内容验证算法

```python
def verify_knowledge(self, knowledge_id: int, content_hash: str) -> bool:
    transaction = self.get_transaction_by_knowledge_id(knowledge_id)
    if not transaction:
        return False
    
    return transaction.content_hash == content_hash
```

**验证逻辑**：
1. 根据 `knowledge_id` 查找交易
2. 比对交易中的 `content_hash` 与提供的哈希值
3. 完全匹配则验证通过

## 4. 业务流程设计

### 4.1 存证流程


```
1. 接收存证请求
   ├─ 参数：type, knowledge_id, user_id, content_hash, metadata
   └─ 验证：必填字段检查

2. 创建交易对象
   ├─ 生成交易ID
   ├─ 记录时间戳
   └─ 验证交易有效性

3. 添加到待处理队列
   └─ blockchain.pending_transactions.append(transaction)

4. 创建新区块
   ├─ 打包所有待处理交易
   ├─ 设置 index = len(chain)
   ├─ 设置 previous_hash = latest_block.hash
   └─ 计算当前区块哈希

5. 添加到区块链
   └─ blockchain.chain.append(block)

6. 持久化到数据库（异步）
   └─ db_manager.save_block(block)

7. 返回结果
   └─ block_index, transaction_id, status, timestamp
```

### 4.2 验证流程

```
1. 接收验证请求
   └─ 参数：knowledge_id, content_hash

2. 查找交易
   └─ blockchain.get_transaction_by_knowledge_id(knowledge_id)

3. 比对哈希值
   ├─ 交易存在？
   │  ├─ 是：比对 content_hash
   │  └─ 否：返回"未找到"
   └─ 哈希匹配？
      ├─ 是：验证通过
      └─ 否：验证失败

4. 查找区块位置
   └─ 遍历区块链，找到交易所在区块

5. 返回验证结果
   └─ is_valid, block_index, timestamp, message
```

### 4.3 证书生成流程

```
1. 接收证书请求
   └─ 参数：knowledge_id, knowledge_title, user_id, user_name

2. 查找区块链记录
   ├─ 查找交易
   └─ 查找区块

3. 生成证书数据
   ├─ 生成证书编号
   ├─ 生成验证URL
   └─ 创建 CertificateData 对象

4. 生成二维码
   ├─ 编码验证URL
   ├─ 生成 PNG 图片
   └─ 保存到 qrcodes/ 目录

5. 生成 PDF 证书
   ├─ 绘制证书头部
   ├─ 绘制证书信息
   ├─ 嵌入二维码
   ├─ 添加水印
   └─ 保存到 certificates/ 目录

6. 返回证书信息
   └─ certificate_id, pdf_url, qr_code_url, verification_url
```

## 7. 安全设计

### 7.1 数据完整性

**哈希链保护**：
```
Block 0 (Genesis)
  hash: 0x1234...
  previous_hash: 0x0000...

Block 1
  hash: 0x5678...
  previous_hash: 0x1234...  ← 指向 Block 0

Block 2
  hash: 0x9abc...
  previous_hash: 0x5678...  ← 指向 Block 1
```

**篡改检测**：
- 修改 Block 1 的内容 → Block 1 的 hash 改变
- Block 2 的 previous_hash 不再匹配 → 验证失败
- 整条链从 Block 1 开始全部失效

### 7.2 内容防伪

**SHA-256 特性**：
- 单向函数：无法从哈希值反推内容
- 抗碰撞：几乎不可能找到两个内容产生相同哈希
- 确定性：相同内容总是产生相同哈希

**应用**：
- 存储哈希值而非完整内容（隐私保护）
- 验证时重新计算哈希并比对
- 任何内容修改都会导致哈希值变化

### 7.3 时间戳证明

**不可篡改的时间证明**：
- 交易创建时记录 `timestamp`
- 区块创建时记录 `timestamp`
- 时间戳写入区块后无法修改
- 可作为版权纠纷的时间证据
