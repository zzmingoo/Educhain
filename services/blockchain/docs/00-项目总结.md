# EduChain 区块链存证服务 - 项目总结

## 📋 项目概况

### 项目名称
EduChain Blockchain Service - 教育知识共享平台区块链存证服务

### 项目定位
本模块是 EduChain 教育知识共享平台的核心技术模块，专注于为教育内容提供基于区块链技术的存证、验证和证书生成服务。

### 技术特点
- ✅ **合规性**：不涉及虚拟货币、代币或挖矿，专注数据存证
- ✅ **高性能**：即时确认，无需等待共识，响应时间 < 100ms
- ✅ **易集成**：标准 REST API，完整的接口文档
- ✅ **可扩展**：模块化设计，支持自定义扩展
- ✅ **生产就绪**：Docker 容器化，完整的部署方案

## 🎯 核心功能

### 1. 内容存证
将教育内容的关键信息（内容哈希、作者、时间戳）记录到区块链，形成不可篡改的存证记录。

**API**: `POST /api/blockchain/certify`

**应用场景**：
- 用户发布原创教育内容
- 记录内容修改历史
- 版权保护和时间戳证明

### 2. 内容验证
通过比对区块链上的内容哈希值，验证内容的真实性和完整性。

**API**: `POST /api/blockchain/verify`

**应用场景**：
- 验证内容是否被篡改
- 确认内容发布时间
- 版权纠纷时提供证据

### 3. 证书生成
为已存证的内容自动生成专业的 PDF 证书，包含二维码验证功能。

**API**: `POST /api/blockchain/certificates`

**证书特点**：
- 专业设计（A4 格式，水印防伪）
- 二维码验证（扫码即可验证）
- 区块链背书（不可伪造）
- 永久有效（区块链记录永久保存）

### 4. 区块链浏览
提供完整的区块链浏览功能，支持查询区块、交易和统计信息。

**API**: 
- `GET /api/blockchain/block/{index}` - 查询区块
- `GET /api/blockchain/transaction/{knowledge_id}` - 查询交易
- `GET /api/blockchain/stats` - 统计信息

## 🔒 安全设计

### 数据安全
- **SHA-256 哈希**：确保内容完整性
- **哈希链保护**：防止历史数据篡改
- **只存储哈希**：保护内容隐私

### 接口安全
- **输入验证**：Pydantic 模型验证
- **SQL 注入防护**：ORM 参数化查询
- **CORS 配置**：限制跨域访问
- **请求限流**：防止 DDoS 攻击

### 传输安全
- **HTTPS**：加密传输
- **响应压缩**：GZip 压缩
- **安全头部**：防止常见攻击

## 🚀 快速开始

### 本地开发

```bash
# 克隆项目
git clone https://github.com/zzmingoo/educhain.git
cd educhain/services/blockchain

# 安装依赖
pip install -r requirements.txt

# 启动服务
python main.py

# 访问 API 文档
open http://localhost:8000/docs
```

### Docker 部署

```bash
# 构建镜像
docker build -t educhain-blockchain:latest .

# 运行容器
docker run -d -p 8000:8000 educhain-blockchain:latest

# 查看日志
docker logs -f blockchain-service
```

### Docker Compose

```bash
# 启动所有服务（包括数据库）
docker-compose up -d

# 查看状态
docker-compose ps

# 停止服务
docker-compose down
```
