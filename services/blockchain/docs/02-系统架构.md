### 1.2 分层说明

#### API 层
- 负责接收 HTTP 请求
- 参数验证和错误处理
- 响应格式化
- CORS 跨域处理

#### 业务逻辑层
- 区块链核心逻辑
- 交易处理和验证
- 证书生成业务
- 数据验证和完整性检查

#### 数据层
- 数据模型定义
- 对象关系映射
- 数据序列化/反序列化

#### 持久化层
- 数据库连接管理
- CRUD 操作
- 事务管理
- 数据迁移

## 2. 核心模块

### 2.1 模块划分

```
services/blockchain/
├── app/
│   ├── __init__.py           # 应用初始化
│   ├── api.py                # API 路由和端点
│   ├── config.py             # 配置管理
│   ├── blockchain.py         # 区块链核心
│   ├── block.py              # 区块定义
│   ├── transaction.py        # 交易定义
│   ├── certificate.py        # 证书数据
│   ├── certificate_pdf.py    # PDF 生成
│   ├── qr_code.py           # 二维码生成

│       ├── __init__.py
```

### 2.2 模块职责

| 模块 | 职责 | 依赖 |
|------|------|------|
| api.py | HTTP 接口、路由、请求处理 | blockchain, certificate, db_manager |
| blockchain.py | 区块链逻辑、链管理、验证 | block, transaction |
| block.py | 区块数据结构、哈希计算 | transaction |
| transaction.py | 交易数据结构、验证 | 无 |
| certificate.py | 证书数据生成、验证 | 无 |
| certificate_pdf.py | PDF 文件生成 | certificate |
| qr_code.py | 二维码图片生成 | 无 |
| db_manager.py | 数据库操作、持久化 | block, transaction |
| config.py | 配置管理、环境变量 | 无 |

## 4. 数据流

### 4.1 存证流程

```
客户端请求
    ↓
[POST /api/blockchain/certify]
    ↓
API 层：参数验证
    ↓
创建 Transaction 对象
    ↓
验证交易有效性
    ↓
添加到待处理队列
    ↓
创建新区块（打包交易）
    ↓
计算区块哈希
    ↓
添加到区块链
    ↓
保存到数据库（异步）
    ↓
返回区块信息
    ↓
客户端收到响应
```

### 4.2 验证流程

```
客户端请求（knowledge_id + content_hash）
    ↓
[POST /api/blockchain/verify]
    ↓
根据 knowledge_id 查找交易
    ↓
比对 content_hash
    ↓
查找交易所在区块
    ↓
返回验证结果
    ↓
客户端收到响应
```

### 4.3 证书生成流程

```
客户端请求
    ↓
[POST /api/blockchain/certificates]
    ↓
查找知识内容的交易
    ↓
查找交易所在区块
    ↓
创建证书数据对象
    ↓
生成验证 URL
    ↓
生成二维码图片
    ↓
生成 PDF 证书
    ↓
返回证书信息和下载链接
    ↓
客户端收到响应
```

## 5. 接口设计

### 5.1 RESTful 设计原则

- 使用标准 HTTP 方法（GET, POST, PUT, DELETE）
- 资源导向的 URL 设计
- 统一的响应格式
- 合理的状态码使用

### 5.2 API 分组

#### 区块链操作
- `POST /api/blockchain/certify` - 存证
- `POST /api/blockchain/verify` - 验证
- `POST /api/blockchain/create-block` - 创建区块
- `GET /api/blockchain/chain` - 获取链信息

#### 查询接口
- `GET /api/blockchain/block/{index}` - 查询区块
- `GET /api/blockchain/transaction/{knowledge_id}` - 查询交易
- `GET /api/blockchain/user/{user_id}/transactions` - 用户交易
- `GET /api/blockchain/stats` - 统计信息

#### 证书接口
- `POST /api/blockchain/certificates` - 生成证书
- `GET /api/blockchain/certificates/{id}/download` - 下载证书
- `GET /api/blockchain/certificates/{id}/verify` - 验证证书
- `GET /api/blockchain/certificates/knowledge/{id}` - 查询证书

#### 系统接口
- `GET /` - 服务信息
- `GET /health` - 健康检查

## 7. 性能设计

- **即时确认**：无需等待共识
- **数据库索引**：优化查询性能
- **后台任务**：数据库写入异步化

### 7.2 性能指标

| 存证 | < 100ms | 1000+ req/s |
| 验证 | < 50ms | 2000+ req/s |
| 查询 | < 30ms | 3000+ req/s |
| 证书生成 | < 500ms | 100+ req/s |

## 10. 总结

本模块采用分层架构设计，职责清晰、易于维护。通过 FastAPI 提供高性能的 REST API，使用 SQLAlchemy 实现数据持久化，支持 Docker 容器化部署。架构设计兼顾了性能、安全性和可扩展性，为教育内容存证提供了可靠的技术支撑。
