## 2. 项目启动

### 2.1 克隆项目

```bash
git clone https://github.com/zzmingoo/educhain.git
cd educhain/apps/web-legacy
```

### 2.2 安装依赖

```bash
npm install
```

### 2.3 环境配置

复制环境变量文件：

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```env
# API 基础地址
VITE_API_BASE_URL=http://localhost:8080/api

# 应用标题
VITE_APP_TITLE=EduChain

# 是否启用 Mock
VITE_USE_MOCK=false
```

### 2.4 启动开发服务器

```bash
# 连接真实后端
npm run dev

# 使用 Mock 数据
npm run dev:mock
```

访问地址: http://localhost:5173

## 3. 开发模式

### 3.1 真实后端模式

连接到本地或远程后端 API：

```bash
npm run dev
```

需要确保后端服务已启动（默认端口 8080）。

### 3.2 Mock 数据模式

使用 MSW 模拟后端 API：

```bash
npm run dev:mock
```

或者设置环境变量：

```env
VITE_USE_MOCK=true
```

Mock 数据位于 `src/mock/` 目录。

## 4. 项目结构说明

### 4.1 添加新页面

1. 在 `src/pages/` 创建页面组件
2. 在 `src/router/index.tsx` 添加路由
3. 在 `src/constants/routes.ts` 添加路由常量

示例：

```typescript
// src/pages/NewPage.tsx
export const NewPage = () => {
  return <div>New Page</div>;
};

// src/router/index.tsx
{
  path: '/new-page',
  element: <NewPage />
}

// src/constants/routes.ts
export const ROUTES = {
  NEW_PAGE: '/new-page',
};
```

### 4.2 添加新组件

在 `src/components/` 对应目录创建组件：

```typescript
// src/components/common/MyComponent.tsx
import { FC } from 'react';

interface MyComponentProps {
  title: string;
}

export const MyComponent: FC<MyComponentProps> = ({ title }) => {
  return <div>{title}</div>;
};
```

### 4.3 添加 API 服务

在 `src/services/` 创建服务文件：

```typescript
// src/services/myService.ts
import { apiClient } from './api';

export const myService = {
  getList: () => apiClient.get('/my-endpoint'),
  getDetail: (id: number) => apiClient.get(`/my-endpoint/${id}`),
  create: (data: any) => apiClient.post('/my-endpoint', data),
};
```

### 4.4 添加 Mock 数据

在 `src/mock/` 添加 Mock 处理器：

```typescript
// src/mock/handlers/myHandlers.ts
import { rest } from 'msw';

export const myHandlers = [
  rest.get('/api/my-endpoint', (req, res, ctx) => {
    return res(
      ctx.json({
        code: 200,
        data: [],
        message: '成功'
      })
    );
  }),
];

// 在 src/mock/server.ts 中导入
import { myHandlers } from './handlers/myHandlers';
export const handlers = [...myHandlers, ...otherHandlers];
```

## 5. 开发规范

### 5.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件文件 | PascalCase | `UserProfile.tsx` |
| 工具文件 | camelCase | `formatDate.ts` |
| 常量 | UPPER_SNAKE_CASE | `API_BASE_URL` |
| 接口/类型 | PascalCase | `UserInfo` |
| 变量/函数 | camelCase | `getUserInfo` |

### 5.2 组件规范

```typescript
// 推荐的组件结构
import { FC, useState, useEffect } from 'react';
import { Button } from 'antd';
import styles from './MyComponent.module.css';

// 1. 类型定义
interface MyComponentProps {
  title: string;
  onSubmit?: () => void;
}

// 2. 组件定义
export const MyComponent: FC<MyComponentProps> = ({ title, onSubmit }) => {
  // 3. Hooks
  const [count, setCount] = useState(0);
  
  useEffect(() => {
    // 副作用
  }, []);
  
  // 4. 事件处理函数
  const handleClick = () => {
    setCount(count + 1);
    onSubmit?.();
  };
  
  // 5. 渲染
  return (
    <div className={styles.container}>
      <h1>{title}</h1>
      <Button onClick={handleClick}>Count: {count}</Button>
    </div>
  );
};
```

### 5.3 TypeScript 规范

```typescript
// ✅ 推荐：明确的类型定义
interface User {
  id: number;
  name: string;
  email: string;
}

const getUser = (id: number): Promise<User> => {
  return apiClient.get(`/users/${id}`);
};

// ❌ 避免：使用 any
const getData = (): any => { ... };

// ✅ 推荐：使用泛型
const getData = <T>(): Promise<T> => { ... };
```

## 8. 常见问题

### 8.1 端口被占用

```bash
# 查找占用端口的进程
lsof -i :5173

# 杀死进程
kill -9 <PID>

# 或者修改端口
vite --port 3000
```

### 8.2 依赖安装失败

```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 8.3 类型错误

```bash
# 重新生成类型
npm run type-check

# 重启 TypeScript 服务器（VS Code）
Cmd/Ctrl + Shift + P → TypeScript: Restart TS Server
```

### 8.4 Mock 不生效

检查：

1. 是否设置了 `VITE_USE_MOCK=true`
2. `public/mockServiceWorker.js` 是否存在
3. 浏览器控制台是否有 MSW 启动日志

## 9. 性能优化建议

### 9.1 组件优化

```typescript
// 使用 memo 避免不必要的重渲染
export const ExpensiveComponent = memo(({ data }: Props) => {
  // ...
});

// 使用 useMemo 缓存计算结果
const sortedList = useMemo(() => {
  return list.sort((a, b) => a.id - b.id);
}, [list]);

// 使用 useCallback 缓存函数
const handleClick = useCallback(() => {
  console.log('clicked');
}, []);
```

### 9.2 懒加载

```typescript
// 路由懒加载
const AdminPage = lazy(() => import('./pages/admin/Dashboard'));

// 组件懒加载
<Suspense fallback={<Loading />}>
  <AdminPage />
</Suspense>
```

### 9.3 图片优化

```typescript
// 使用 loading="lazy"
<img src={url} alt="..." loading="lazy" />

// 使用 WebP 格式
<picture>
  <source srcSet="image.webp" type="image/webp" />
  <img src="image.jpg" alt="..." />
</picture>
```

## 10. Git 工作流

### 10.1 分支管理

```bash
# 创建功能分支
git checkout -b feature/my-feature

# 提交代码
git add .
git commit -m "feat: add new feature"

# 推送到远程
git push origin feature/my-feature
```

### 10.2 提交规范

使用 Conventional Commits：

```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式调整
refactor: 代码重构
test: 测试相关
chore: 构建/工具相关
```

示例：

```bash
git commit -m "feat: add user profile page"
git commit -m "fix: resolve login issue"
git commit -m "docs: update README"
```
