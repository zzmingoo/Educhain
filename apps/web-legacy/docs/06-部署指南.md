## 1. 构建准备

### 1.1 环境要求

| 软件 | 版本要求 |
|------|----------|
| Node.js | >= 18.0.0 |
| npm | >= 9.0.0 |

### 1.2 环境变量配置

创建生产环境配置文件 `.env.production`:

```env
# API 基础地址
VITE_API_BASE_URL=https://api.educhain.cc/api

# 应用标题
VITE_APP_TITLE=EduChain

# 是否启用 Mock（生产环境应为 false）
VITE_USE_MOCK=false
```

## 2. 本地构建

### 2.1 安装依赖

```bash
cd apps/web-legacy
npm install
```

### 2.2 执行构建

```bash
# 生产构建
npm run build

npm run build:mock
```


访问 http://localhost:4173 查看构建结果。

```

├── assets/
│   ├── index-[hash].js      # 主 JS 文件

└── favicon.ico              # 网站图标
```

## 3. 静态服务器部署

### 3.1 Nginx 部署

#### 3.1.1 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nginx

# macOS
brew install nginx
```

#### 3.1.2 配置 Nginx

创建配置文件 `/etc/nginx/sites-available/educhain-legacy`:

```nginx
server {
    listen 80;
    server_name legacy.educhain.cc;
    
    root /var/www/educhain-legacy;
    index index.html;
    
    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 1000;
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API 代理（可选）
    location /api/ {
        proxy_pass http://localhost:8080/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### 3.1.3 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/educhain-legacy /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

#### 3.1.4 部署文件

```bash
# 上传构建产物
scp -r dist/* user@server:/var/www/educhain-legacy/

# 或使用 rsync
rsync -avz --delete dist/ user@server:/var/www/educhain-legacy/
```

### 3.2 Apache 部署

#### 3.2.1 配置 Apache

创建配置文件 `/etc/apache2/sites-available/educhain-legacy.conf`:

```apache
<VirtualHost *:80>
    ServerName legacy.educhain.cc
    DocumentRoot /var/www/educhain-legacy
    
    <Directory /var/www/educhain-legacy>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
        
        # SPA 路由支持
        RewriteEngine On
        RewriteBase /
        RewriteRule ^index\.html$ - [L]
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteCond %{REQUEST_FILENAME} !-d
        RewriteRule . /index.html [L]
    </Directory>
    
    # 启用压缩
    <IfModule mod_deflate.c>
        AddOutputFilterByType DEFLATE text/html text/plain text/xml text/css text/javascript application/javascript application/json
    </IfModule>
    
    # 静态资源缓存
    <FilesMatch "\.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$">
        Header set Cache-Control "max-age=31536000, public, immutable"
    </FilesMatch>
</VirtualHost>
```

#### 3.2.2 启用配置

```bash
# 启用站点
sudo a2ensite educhain-legacy

# 启用必要的模块
sudo a2enmod rewrite
sudo a2enmod deflate
sudo a2enmod headers

# 重启 Apache
sudo systemctl restart apache2
```

## 4. CDN 部署

### 4.1 Cloudflare Pages

#### 4.1.1 连接 GitHub 仓库

1. 登录 Cloudflare Dashboard
2. 进入 Pages
3. 连接 GitHub 仓库

#### 4.1.2 配置构建

```yaml
Build command: npm run build:mock
Build output directory: dist
Root directory: apps/web-legacy
```

#### 4.1.3 环境变量

在 Cloudflare Pages 设置中添加：

```
VITE_API_BASE_URL=https://api.educhain.cc/api
VITE_USE_MOCK=true
```

#### 4.1.4 自动部署

推送到 GitHub 后自动触发部署。

### 4.2 Vercel 部署

#### 4.2.1 安装 Vercel CLI

```bash
npm install -g vercel
```

#### 4.2.2 部署

```bash
cd apps/web-legacy
vercel --prod
```

#### 4.2.3 配置文件

创建 `vercel.json`:

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

### 4.3 Netlify 部署

#### 4.3.1 配置文件

创建 `netlify.toml`:

```toml
[build]
  command = "npm run build"
  publish = "dist"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

[[headers]]
  for = "/assets/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"
```

#### 4.3.2 部署

```bash
# 安装 Netlify CLI
npm install -g netlify-cli

# 部署
netlify deploy --prod
```

## 5. Docker 部署

### 5.1 Dockerfile

创建 `Dockerfile`:

```dockerfile
# 构建阶段
FROM node:18-alpine AS builder

WORKDIR /app

# 复制依赖文件
COPY package*.json ./
RUN npm ci

# 复制源代码
COPY . .

# 构建
RUN npm run build

# 生产阶段
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制 Nginx 配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 5.2 Nginx 配置

创建 `nginx.conf`:

```nginx
server {
    listen 80;
    server_name _;
    
    root /usr/share/nginx/html;
    index index.html;
    
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 5.3 构建和运行

```bash
# 构建镜像
docker build -t educhain-legacy:latest .

# 运行容器
docker run -d -p 80:80 --name educhain-legacy educhain-legacy:latest
```

### 5.4 Docker Compose

创建 `docker-compose.yml`:

```yaml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "80:80"
    restart: unless-stopped
    environment:
      - NODE_ENV=production
```

运行：

```bash
docker-compose up -d
```

## 8. 安全配置

### 8.1 HTTPS 配置

使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d legacy.educhain.cc

# 自动续期
sudo certbot renew --dry-run
```

### 8.2 安全头

Nginx 安全头配置：

```nginx
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "no-referrer-when-downgrade" always;
```

## 10. 故障排查

### 10.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 404 错误 | 路由配置错误 | 检查 Nginx/Apache 配置 |
| 白屏 | JS 加载失败 | 检查构建产物和路径 |
| API 请求失败 | CORS 或代理配置 | 检查后端 CORS 设置 |

### 10.2 日志查看

```bash
# Nginx 日志
tail -f /var/log/nginx/educhain-legacy-error.log

# Docker 日志
docker logs -f educhain-legacy
```
