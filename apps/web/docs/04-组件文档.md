## 组件概述

### 组件分类

EduChain Web 前端应用包含 80+ 个组件，按功能分为以下几类：

**1. 布局组件 (Layout Components)**
- 页面整体布局
- 导航栏和侧边栏
- 页脚和面包屑
- 位置：`components/layout/`

**2. 通用组件 (Common Components)**
- 可复用的基础组件
- 跨模块使用
- 位置：`components/common/`

**3. 业务组件 (Business Components)**
- 特定功能模块组件
- 知识库、区块链、管理后台等
- 位置：`components/admin/`, `components/knowledge/`, `components/blockchain/`

**4. UI 基础组件 (UI Components)**
- 最基础的 UI 元素
- 按钮、输入框、卡片等
- 位置：`components/ui/`

**5. Provider 组件 (Provider Components)**
- 全局状态和服务提供者
- Context 和 Mock 服务
- 位置：`components/providers/`

### 组件命名规范

- 组件文件使用 PascalCase：`UserCard.tsx`
- 组件名称与文件名一致
- 样式文件同名：`UserCard.css`
- 测试文件添加后缀：`UserCard.test.tsx`

## 布局组件

### Navbar - 导航栏

**功能说明**

顶部导航栏组件，提供全局导航和用户操作入口。

**主要功能**
- 显示应用 Logo 和名称
- 主导航菜单（首页、知识库、区块链、社区等）
- 搜索框快速入口
- 用户头像和下拉菜单
- 语言切换器
- 主题切换器
- 通知中心入口

**使用场景**
- 所有页面的顶部
- 固定定位，始终可见

**响应式设计**
- 桌面端：完整显示所有功能
- 移动端：折叠为汉堡菜单

### Sidebar - 侧边栏

**功能说明**

侧边栏组件，用于管理后台和用户中心的二级导航。

**主要功能**
- 显示当前模块的子菜单
- 支持菜单折叠/展开
- 高亮当前激活菜单项
- 显示菜单图标和徽章

**使用场景**
- 管理后台页面
- 用户中心页面

**状态管理**
- 折叠状态保存到 LocalStorage
- 支持快捷键切换（Ctrl + B）

### Footer - 页脚

**功能说明**

页面底部组件，显示版权信息和链接。

**主要内容**
- 版权声明
- 关于我们、联系方式
- 法律条款链接
- 社交媒体链接
- 备案信息

**使用场景**
- 所有页面的底部
- 静态内容，不随页面滚动

### Breadcrumb - 面包屑

**功能说明**

面包屑导航组件，显示当前页面在网站结构中的位置。

**主要功能**
- 显示页面层级路径
- 支持点击跳转到上级页面
- 自动根据路由生成
- 支持自定义面包屑项

**使用场景**
- 详情页面
- 多层级页面

## 通用组件

### Button - 按钮

**功能说明**

通用按钮组件，支持多种样式和状态。

**变体类型**
- Primary：主要操作按钮
- Secondary：次要操作按钮
- Outline：轮廓按钮
- Ghost：幽灵按钮
- Link：链接样式按钮

**尺寸规格**
- Small：小尺寸（高度 32px）
- Medium：中等尺寸（高度 40px，默认）
- Large：大尺寸（高度 48px）

**状态支持**
- 默认状态
- 悬停状态
- 激活状态
- 禁用状态
- 加载状态

### Input - 输入框

**功能说明**

通用输入框组件，支持多种输入类型。

**输入类型**
- Text：文本输入
- Password：密码输入
- Email：邮箱输入
- Number：数字输入
- Search：搜索输入
- Textarea：多行文本

**功能特性**
- 前缀和后缀图标
- 清除按钮
- 字符计数
- 错误提示
- 禁用状态
- 只读状态

### Modal - 模态框

**功能说明**

模态对话框组件，用于显示重要信息或表单。

**功能特性**
- 遮罩层背景
- 关闭按钮
- 标题和内容区域
- 底部操作按钮
- 支持拖拽（可选）
- 键盘 ESC 关闭
- 点击遮罩关闭（可选）

**尺寸规格**
- Small：小尺寸（宽度 400px）
- Medium：中等尺寸（宽度 600px，默认）
- Large：大尺寸（宽度 800px）
- Full：全屏

### Loading - 加载状态

**功能说明**

加载状态指示器，显示数据加载中。

**显示类型**
- Spinner：旋转加载器
- Skeleton：骨架屏
- Progress：进度条
- Overlay：全屏遮罩

**使用场景**
- 页面初始加载
- 数据请求中
- 文件上传中
- 操作处理中

### ErrorBoundary - 错误边界

**功能说明**

React 错误边界组件，捕获子组件错误并显示友好提示。

**功能特性**
- 捕获渲染错误
- 显示错误信息
- 提供重试按钮
- 错误日志上报
- 降级 UI 显示

**使用场景**
- 包裹整个应用
- 包裹关键模块
- 包裹第三方组件

### MockIndicator - Mock 指示器

**功能说明**

Mock 模式指示器，显示当前是否使用 Mock 数据。

**显示内容**
- Mock 模式标识
- 当前 API 地址
- 切换模式按钮（开发环境）

**使用场景**
- 开发环境显示
- 生产环境隐藏

## Provider 组件

### MockProvider - Mock 服务提供者

**功能说明**

Mock 服务初始化和管理组件，负责启动 MSW。

**主要功能**
- 检测 Mock 模式
- 初始化 MSW Worker
- 显示加载状态

**使用位置**
- 应用根布局

3. 启动 MSW Service Worker
4. 标记初始化完成
5. 渲染子组件

## 组件开发规范

### 组件结构

**推荐结构**
1. 导入语句
2. 类型定义（Props、State）
3. 组件函数
4. 样式定义（如使用 CSS-in-JS）
5. 导出语句

### Props 设计原则

**命名规范**
- 使用描述性名称
- 布尔值以 `is`、`has`、`should` 开头
- 事件处理器以 `on` 开头
- 渲染函数以 `render` 开头

**类型定义**
- 所有 Props 必须定义 TypeScript 类型
- 使用 `interface` 定义 Props
- 标注可选属性
- 提供默认值

### 性能优化

**优化技巧**
- 使用 `React.memo` 避免不必要的重渲染
- 使用 `useMemo` 缓存计算结果
- 使用 `useCallback` 缓存函数引用
- 避免在渲染函数中创建新对象
- 合理使用 `key` 属性

### 可访问性

**ARIA 属性**
- 添加适当的 `aria-label`
- 使用语义化 HTML 标签
- 支持键盘导航
- 提供焦点指示
- 确保颜色对比度

### 测试要求

**测试覆盖**
- 组件渲染测试
- Props 传递测试
- 事件处理测试
- 边界情况测试
- 快照测试（可选）

## 组件使用最佳实践

### 1. 合理拆分组件

将大组件拆分为小组件，提高可维护性和复用性。

### 2. 避免 Props 钻取

使用 Context 或状态管理库传递深层数据，避免多层 Props 传递。

### 3. 保持组件纯净

组件应该是纯函数，相同的 Props 应该产生相同的输出。

### 4. 使用组合而非继承

通过组合小组件构建复杂组件，而不是使用继承。

### 5. 提供清晰的 API

组件的 Props 应该简单明了，易于理解和使用。

## 总结

EduChain Web 前端应用包含 80+ 个精心设计的组件，涵盖：

- **布局组件**：提供页面整体结构
- **通用组件**：可复用的基础组件
- **业务组件**：特定功能模块组件
- **UI 基础组件**：最基础的 UI 元素
- **Provider 组件**：全局服务提供者

所有组件都遵循统一的开发规范，具有良好的类型定义、性能优化和可访问性支持。
