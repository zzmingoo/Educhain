## 概述

EduChain Web 采用完全静态导出（Static Site Generation, SSG）的方式构建，可以部署到任何支持静态网站托管的平台。本文档主要介绍如何部署到 Cloudflare Pages，这是一个高性能、全球分发的静态网站托管服务。

### 为什么选择 Cloudflare Pages？

1. **全球 CDN**：自动分发到 Cloudflare 的全球 300+ 数据中心
2. **免费额度**：每月 500 次构建，无限带宽和请求
3. **自动 HTTPS**：免费 SSL 证书，自动续期
4. **Git 集成**：支持 GitHub、GitLab 自动部署
5. **预览环境**：每个 PR 自动生成预览链接
6. **边缘计算**：支持 Cloudflare Workers 扩展功能
7. **快速构建**：平均构建时间 < 1 分钟

### 部署架构

```
GitHub Repository
       ↓
Cloudflare Pages (自动构建)
       ↓
全球 CDN 分发
       ↓
用户访问
```

## 静态导出配置

### 1. Next.js 配置

在 `next.config.ts` 中启用静态导出：

```typescript
import type { NextConfig } from "next";
import { withIntlayer } from "next-intlayer/server";

const nextConfig: NextConfig = {
  // 启用静态导出
  output: 'export',
  
  // 禁用图片优化（静态导出不支持）
  images: {
    unoptimized: true,
  },
  
  // 添加 trailing slash（Cloudflare Pages 推荐）
  trailingSlash: true,
  
  // 优化预加载策略
  experimental: {
    optimizePackageImports: ['next-intlayer'],
  },
};

export default withIntlayer(nextConfig);
```

**配置说明**：

- `output: 'export'`：启用静态导出模式，生成纯 HTML/CSS/JS 文件
- `images.unoptimized: true`：禁用 Next.js 的图片优化（静态导出不支持）
- `trailingSlash: true`：URL 末尾添加斜杠，避免重定向问题
- `experimental.optimizePackageImports`：优化包导入，减小打包体积

### 2. 构建脚本

在 `package.json` 中配置构建脚本：

```json
{
  "scripts": {
    "build": "next build",
    "build:mock": "cross-env NEXT_PUBLIC_USE_MOCK=true next build",
    "deploy:cf": "npm run build:mock && wrangler pages deploy out --project-name educhain-web",
    "preview": "npx serve@latest out"
  }
}
```

**脚本说明**：

- `build`：标准构建（连接真实后端）
- `build:mock`：Mock 模式构建（使用 MSW 模拟数据）
- `deploy:cf`：构建并部署到 Cloudflare Pages
- `preview`：本地预览构建结果

### 3. 静态导出限制

由于静态导出的特性，以下 Next.js 功能不可用：

- **不支持的功能**：
  - 服务端渲染（SSR）
  - 增量静态再生成（ISR）
  - API Routes（需要使用 Cloudflare Workers 替代）
  - 图片优化（需要使用外部服务）
  - 中间件（Middleware）

- **替代方案**：
  - SSR → 静态生成（SSG）
  - API Routes → Cloudflare Workers 或外部 API
  - 图片优化 → Cloudflare Images 或 CDN
  - 中间件 → Cloudflare Workers

## 部署流程

### 方式一：通过 Cloudflare Dashboard（推荐）

#### 1. 创建 Cloudflare Pages 项目

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入 "Pages" 页面
3. 点击 "Create a project"
4. 选择 "Connect to Git"

#### 2. 连接 GitHub 仓库

1. 授权 Cloudflare 访问 GitHub
2. 选择 `zzmingoo/educhain` 仓库
3. 选择要部署的分支（通常是 `main` 或 `master`）

#### 3. 配置构建设置

- **项目名称**: `educhain-web`
- **生产分支**: `main`
- **构建命令**: `npm run build:mock`
- **构建输出目录**: `out`
- **根目录**: `apps/web`（如果是 Monorepo）
- **Node.js 版本**: `20.x`（或更高）

#### 4. 设置环境变量

在 "Environment variables" 部分添加：

```
NEXT_PUBLIC_USE_MOCK=true
NEXT_PUBLIC_API_BASE_URL=https://api.educhain.com
```

#### 5. 开始部署

点击 "Save and Deploy"，Cloudflare 会自动：
1. 克隆仓库
2. 安装依赖
3. 执行构建命令
4. 部署到全球 CDN

#### 6. 查看部署结果

构建完成后，会生成一个临时域名：
```
https://educhain-web.pages.dev
```

### 方式二：通过 Wrangler CLI

#### 1. 安装 Wrangler

```bash
npm install -g wrangler
```

#### 2. 登录 Cloudflare

```bash
wrangler login
```

#### 3. 构建项目

```bash
cd apps/web
npm run build:mock
```

#### 4. 部署到 Cloudflare Pages

```bash
wrangler pages deploy out --project-name educhain-web
```

或使用快捷脚本：

```bash
npm run deploy:cf
```

#### 5. 查看部署状态

```bash
wrangler pages deployment list --project-name educhain-web
```

### 方式三：通过 GitHub Actions（CI/CD）

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy to Cloudflare Pages

on:
  push:
    branches:
      - main
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      deployments: write
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
        working-directory: apps/web
      
      - name: Build
        run: npm run build:mock
        working-directory: apps/web
        env:
          NEXT_PUBLIC_USE_MOCK: true
      
      - name: Deploy to Cloudflare Pages
        uses: cloudflare/pages-action@v1
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          projectName: educhain-web
          directory: apps/web/out
          gitHubToken: ${{ secrets.GITHUB_TOKEN }}
```

**配置 Secrets**：

在 GitHub 仓库设置中添加：
- `CLOUDFLARE_API_TOKEN`：Cloudflare API Token
- `CLOUDFLARE_ACCOUNT_ID`：Cloudflare Account ID

## 环境变量配置

### 1. 环境变量类型

Next.js 支持两种类型的环境变量：

- **服务端变量**：只在构建时可用，不会暴露到浏览器
- **客户端变量**：以 `NEXT_PUBLIC_` 开头，会打包到客户端代码

### 2. 项目中的环境变量

```bash
# Mock 模式开关
NEXT_PUBLIC_USE_MOCK=true

# API 基础 URL（真实后端模式）
NEXT_PUBLIC_API_BASE_URL=https://api.educhain.com

# 区块链服务 URL
NEXT_PUBLIC_BLOCKCHAIN_API_URL=https://blockchain.educhain.com

# 分析和监控
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
NEXT_PUBLIC_SENTRY_DSN=https://xxx@sentry.io/xxx
```

### 3. 在 Cloudflare Pages 中设置

#### 通过 Dashboard 设置

1. 进入项目设置页面
2. 点击 "Environment variables"
3. 添加变量（区分生产环境和预览环境）

#### 通过 Wrangler 设置

```bash
wrangler pages secret put NEXT_PUBLIC_API_BASE_URL --project-name educhain-web
```

### 4. 环境变量优先级

```
Cloudflare Pages 环境变量 > .env.production > .env.local > .env
```

## 自定义域名

### 1. 添加自定义域名

1. 进入 Cloudflare Pages 项目设置
2. 点击 "Custom domains"
3. 点击 "Set up a custom domain"
4. 输入域名（如 `educhain.com` 或 `www.educhain.com`）

### 2. DNS 配置

如果域名已在 Cloudflare 管理：
- 自动添加 CNAME 记录
- 自动配置 SSL 证书

如果域名在其他服务商：
- 添加 CNAME 记录指向 `educhain-web.pages.dev`
- 等待 DNS 传播（通常 5-10 分钟）

### 3. SSL/TLS 配置

Cloudflare 自动提供免费的 SSL 证书：
- 支持 TLS 1.2 和 1.3
- 自动续期
- 支持 HSTS

### 4. 重定向规则

在 Cloudflare Pages 设置中配置重定向：

```
# 强制 HTTPS
http://* -> https://:splat 301

# www 重定向到根域名
https://www.educhain.com/* -> https://educhain.com/:splat 301

# 旧路径重定向
/old-path -> /new-path 301
```

## 性能优化

### 1. 构建优化

#### 减小打包体积

- 使用动态导入（Dynamic Import）
- 启用 Tree Shaking
- 移除未使用的依赖
- 压缩图片和资源

#### 优化构建时间

- 使用 Turbopack（Next.js 14+）
- 启用增量构建
- 缓存 node_modules

### 2. 运行时优化

#### 资源加载

- 使用 Cloudflare CDN 缓存
- 启用 Brotli 压缩
- 配置合理的缓存策略

#### 代码分割

- 按路由分割代码
- 按组件懒加载
- 使用 React.lazy() 和 Suspense

### 3. Cloudflare 优化

#### 缓存策略

在 `public/_headers` 文件中配置：

```
/*
  Cache-Control: public, max-age=3600, must-revalidate

/_next/static/*
  Cache-Control: public, max-age=31536000, immutable

/images/*
  Cache-Control: public, max-age=86400
```

#### 压缩配置

在 `public/_redirects` 文件中配置：

```
/*  /index.html  200
```

#### 安全头部

在 `public/_headers` 中添加安全头部：

```
/*
  X-Frame-Options: DENY
  X-Content-Type-Options: nosniff
  Referrer-Policy: strict-origin-when-cross-origin
  Permissions-Policy: camera=(), microphone=(), geolocation=()
```

### 4. 性能监控

使用 Cloudflare Web Analytics：

1. 在 Cloudflare Dashboard 启用 Web Analytics
2. 添加跟踪代码到 `app/layout.tsx`
3. 查看实时性能指标：
   - 页面加载时间
   - 首次内容绘制（FCP）
   - 最大内容绘制（LCP）
   - 累积布局偏移（CLS）

## 常见问题

### 1. 构建失败怎么办？

**检查构建日志**：
- 在 Cloudflare Dashboard 查看详细日志
- 查找错误信息和堆栈跟踪

**常见原因**：
- Node.js 版本不兼容
- 依赖安装失败
- 环境变量缺失
- 构建命令错误

**解决方案**：
- 指定正确的 Node.js 版本
- 检查 `package.json` 中的依赖
- 确认环境变量已设置
- 本地测试构建命令

### 2. 部署后页面 404？

**原因**：
- 路由配置错误
- 输出目录不正确
- 缺少 `_redirects` 文件

**解决方案**：
- 确认 `output: 'export'` 已启用
- 检查输出目录是否为 `out`
- 添加 `public/_redirects` 文件

### 3. 图片无法显示？

**原因**：
- 使用了 Next.js 的图片优化
- 图片路径错误

**解决方案**：
- 设置 `images.unoptimized: true`
- 使用相对路径或绝对路径
- 将图片放在 `public` 目录

### 4. API 请求失败？

**原因**：
- CORS 配置错误
- API 地址不正确
- 环境变量未设置

**解决方案**：
- 在后端配置 CORS
- 检查 `NEXT_PUBLIC_API_BASE_URL`
- 使用 Mock 模式测试

### 5. 如何回滚到之前的版本？

在 Cloudflare Dashboard：
1. 进入 "Deployments" 页面
2. 找到要回滚的版本
3. 点击 "Rollback to this deployment"

或使用 Wrangler：

```bash
wrangler pages deployment list --project-name educhain-web
wrangler pages deployment rollback <deployment-id> --project-name educhain-web
```

### 6. 如何设置预览环境？

Cloudflare Pages 自动为每个分支和 PR 创建预览环境：

- **分支预览**：`https://<branch>.<project>.pages.dev`
- **PR 预览**：`https://<pr-number>.<project>.pages.dev`

可以在 PR 中看到预览链接，方便测试和审查。

### 7. 如何监控部署状态？

**通过 Dashboard**：
- 实时查看构建日志
- 查看部署历史
- 监控性能指标

**通过 Wrangler**：

```bash
# 查看部署列表
wrangler pages deployment list --project-name educhain-web

# 查看部署详情
wrangler pages deployment tail --project-name educhain-web
```

**通过 Webhook**：
- 配置 Webhook 接收部署通知
- 集成到 Slack、Discord 等

### 8. 如何优化首次加载速度？

1. **启用 HTTP/3**：Cloudflare 自动支持
2. **使用 Early Hints**：预加载关键资源
3. **优化字体加载**：使用 `font-display: swap`
4. **减少 JavaScript 体积**：代码分割和懒加载
5. **使用 Service Worker**：缓存静态资源
