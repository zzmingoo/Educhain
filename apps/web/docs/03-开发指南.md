### 环境变量配置

创建 `.env.local` 文件：

```bash
# API 配置
NEXT_PUBLIC_API_BASE_URL=/api

# Mock 模式开关
NEXT_PUBLIC_USE_MOCK=true

# 其他配置（可选）
NEXT_PUBLIC_APP_NAME=EduChain
NEXT_PUBLIC_APP_VERSION=1.0.0
```

**环境变量说明**

| 变量名 | 说明 | 默认值 | 示例 |
|--------|------|--------|------|
| `NEXT_PUBLIC_API_BASE_URL` | API 基础路径 | `/api` | `https://api.educhain.cc` |
| `NEXT_PUBLIC_USE_MOCK` | 是否启用 Mock | `false` | `true` / `false` |
| `NEXT_PUBLIC_APP_NAME` | 应用名称 | `EduChain` | `EduChain` |
| `NEXT_PUBLIC_APP_VERSION` | 应用版本 | `1.0.0` | `1.0.0` |

**注意事项**
- 以 `NEXT_PUBLIC_` 开头的变量会暴露到浏览器
- `.env.local` 文件不会提交到 Git
- 生产环境使用 `.env.production`

### 混合模式开发

**场景**: 部分接口使用 Mock，部分使用真实后端

**实现方式**

```typescript
// src/mock/handlers/index.ts
export const setupMockServer = async () => {
  // 只 Mock 部分接口
  const handlers = [
    ...authHandlers,      // Mock 认证接口
    ...userHandlers,      // Mock 用户接口
    // 其他接口使用真实后端
  ];
  
  const worker = setupWorker(...handlers);
  await worker.start({
    onUnhandledRequest: 'bypass', // 未匹配的请求放行到真实后端
  });
};
```

## 开发规范

### 代码风格

**TypeScript 规范**

```typescript
// ✅ 推荐：使用接口定义类型
interface User {
  id: number;
  username: string;
  email: string;
}

// ✅ 推荐：使用类型别名定义联合类型
type UserRole = 'ADMIN' | 'LEARNER';

// ✅ 推荐：使用可选链和空值合并
const userName = user?.profile?.name ?? 'Anonymous';

// ❌ 避免：使用 any 类型
const data: any = fetchData(); // 不推荐

// ✅ 推荐：使用泛型
const data = fetchData<User>();
```

**React 组件规范**

```typescript
// ✅ 推荐：使用函数组件
export default function UserCard({ user }: { user: User }) {
  return <div>{user.username}</div>;
}

// ✅ 推荐：使用 Props 接口
interface UserCardProps {
  user: User;
  onEdit?: (user: User) => void;
}

export default function UserCard({ user, onEdit }: UserCardProps) {
  return (
    <div onClick={() => onEdit?.(user)}>
      {user.username}
    </div>
  );
}

// ✅ 推荐：使用 memo 优化性能
export default memo(function UserCard({ user }: UserCardProps) {
  return <div>{user.username}</div>;
});
```

**Hooks 使用规范**

```typescript
// ✅ 推荐：自定义 Hook 以 use 开头
export function useUser(userId: number) {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(true);
  
  useEffect(() => {
    fetchUser(userId).then(setUser).finally(() => setLoading(false));
  }, [userId]);
  
  return { user, loading };
}

// ✅ 推荐：使用 useCallback 缓存函数
const handleClick = useCallback(() => {
  console.log('clicked');
}, []);

// ✅ 推荐：使用 useMemo 缓存计算结果
const filteredUsers = useMemo(() => {
  return users.filter(u => u.status === 1);
}, [users]);
```

## 常见问题

### 1. Mock 服务未启动

**问题**: 页面显示 "Mock 服务初始化中"，但一直不消失

**原因**:
- 环境变量未正确设置
- MSW Worker 脚本未生成
- 浏览器不支持 Service Worker

**解决方案**:

```bash
# 1. 检查环境变量
echo $NEXT_PUBLIC_USE_MOCK  # 应该输出 true

# 2. 重新生成 MSW Worker 脚本
npx msw init public/ --save

# 3. 清除浏览器缓存
# Chrome: DevTools -> Application -> Clear storage

# 4. 重启开发服务器
npm run dev:mock
```

### 2. Token 过期问题

**问题**: 频繁跳转到登录页

**原因**:
- Token 已过期
- Token 刷新失败
- LocalStorage 被清除

**解决方案**:

```typescript
// 检查 Token 状态
import { token } from '@/lib/token';

const accessToken = token.getAccessToken();
console.log('Token:', accessToken);
console.log('Is expired:', token.isExpired(accessToken));
console.log('Remaining time:', token.getRemainingTime(accessToken));

// 手动刷新 Token
import { authService } from '@/services/auth';

const refreshToken = token.getRefreshToken();
const res = await authService.refreshToken(refreshToken);
token.setTokens(res.data.accessToken, res.data.refreshToken);
```

### 3. 国际化内容不显示

**问题**: 页面显示 `[object Object]` 或空白

**原因**:
- 内容文件未正确导出
- 使用方式错误
- Intlayer 配置问题

**解决方案**:

```typescript
// ❌ 错误用法
const content = useIntlayer('page-name');
return <h1>{content.title}</h1>; // 错误：缺少 .value

// ✅ 正确用法
const content = useIntlayer('page-name');
return <h1>{content.title.value}</h1>; // 正确

// 检查内容文件
// page.content.ts
export default {
  key: 'page-name', // 必须与 useIntlayer 参数一致
  content: {
    title: t({
      'zh-CN': '标题',
      en: 'Title',
    }),
  },
} satisfies DeclarationContent;
```

### 4. 样式不生效

**问题**: Tailwind CSS 类名不生效

**原因**:
- 类名拼写错误
- 类名动态生成
- Tailwind 配置问题

**解决方案**:

```typescript
// ❌ 错误：动态类名
const color = 'red';
<div className={`text-${color}-500`}>Text</div>

// ✅ 正确：完整类名
<div className={color === 'red' ? 'text-red-500' : 'text-blue-500'}>
  Text
</div>

// ✅ 正确：使用 clsx
import clsx from 'clsx';

<div className={clsx('text-base', {
  'text-red-500': isError,
  'text-green-500': isSuccess,
})}>
  Text
</div>
```

### 5. 构建失败

**问题**: `npm run build` 报错

**常见错误**:

```bash
# 错误 1: 类型错误
Type 'string | undefined' is not assignable to type 'string'

# 解决：添加类型守卫
const value = data?.value ?? '';

# 错误 2: 模块未找到
Module not found: Can't resolve '@/components/Button'

# 解决：检查路径和文件名
# 确保文件存在且路径正确

# 错误 3: 内存不足
JavaScript heap out of memory

# 解决：增加内存限制
NODE_OPTIONS=--max_old_space_size=4096 npm run build
```

### 6. 路由跳转问题

**问题**: 页面跳转后状态丢失

**原因**:
- 使用 `<a>` 标签而非 `<Link>`
- 状态未持久化

**解决方案**:

```typescript
// ❌ 错误：使用 a 标签
<a href="/knowledge">知识库</a>

// ✅ 正确：使用 Link 组件
import Link from 'next/link';
<Link href="/knowledge">知识库</Link>

// ✅ 正确：使用 useRouter
import { useRouter } from 'next/navigation';

const router = useRouter();
router.push('/knowledge');

// 持久化状态
import { storage } from '@/lib/storage';

// 保存状态
storage.set('draft', draftData);

// 恢复状态
const draft = storage.get('draft');
```

**问题**: 请求返回 404 或 500

**调试步骤**:

```bash
# 3. 检查请求 URL

# 5. 检查后端服务
# 确保后端服务已启动且可访问
curl http://localhost:8080/api/health
```

### 8. 热更新不工作

**问题**: 修改代码后页面不自动刷新

**解决方案**:

```bash
# 1. 重启开发服务器
Ctrl + C
npm run dev

# 2. 清除 .next 缓存
rm -rf .next
npm run dev

# 3. 检查文件监听限制（Linux/Mac）
# 增加文件监听数量
echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

# 4. 使用 --turbo 模式（实验性）
npm run dev -- --turbo
```
