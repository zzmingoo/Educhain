## 概述

EduChain Web 采用 [Intlayer](https://intlayer.org/) 作为国际化解决方案，支持中文（简体）和英文两种语言。Intlayer 是一个专为 Next.js 设计的现代化国际化框架，提供了类型安全、开发友好的 API。

### 为什么选择 Intlayer？

相比传统的 i18n 方案（如 next-i18next、react-intl），Intlayer 具有以下优势：

1. **类型安全**：自动生成 TypeScript 类型定义，编译时检查翻译键
2. **开发体验**：内置可视化编辑器，支持实时预览和在线编辑
3. **Next.js 深度集成**：原生支持 App Router、Server Components、静态导出
4. **零配置路由**：自动处理语言路由（/zh、/en），无需手动配置
5. **性能优化**：按需加载翻译文件，支持代码分割和懒加载

### 支持的语言

- **中文（简体）**：`zh-CN`（默认语言）
- **英文**：`en`

## Intlayer 简介

### 核心概念

Intlayer 的国际化方案基于以下核心概念：

#### 1. 内容声明文件（Content Declaration）

每个组件或页面可以创建一个 `.content.ts` 文件，声明该模块需要的所有翻译内容：

```typescript
// src/components/Header.content.ts
import { t, type DeclarationContent } from "intlayer";

const headerContent = {
  key: "header",
  content: {
    title: t({
      "zh-CN": "教育链",
      en: "EduChain",
    }),
    nav: {
      home: t({
        "zh-CN": "首页",
        en: "Home",
      }),
      knowledge: t({
        "zh-CN": "知识库",
        en: "Knowledge",
      }),
    },
  },
} satisfies DeclarationContent;

export default headerContent;
```

#### 2. 类型安全的翻译钩子

在组件中使用 `useIntlayer` 钩子获取翻译内容，享受完整的类型提示：

```typescript
import { useIntlayer } from "next-intlayer";

export default function Header() {
  const { title, nav } = useIntlayer("header");
  
  return (
    <header>
      <h1>{title}</h1>
      <nav>
        <a href="/">{nav.home}</a>
        <a href="/knowledge">{nav.knowledge}</a>
      </nav>
    </header>
  );
}
```

#### 3. 自动生成的类型定义

Intlayer 会自动扫描所有 `.content.ts` 文件，生成类型定义到 `.intlayer/types/` 目录，确保翻译键的类型安全。

#### 4. 可视化编辑器

Intlayer 提供了一个可视化编辑器，可以在浏览器中实时编辑翻译内容：

```bash
npm run start:editor
```

编辑器支持：
- 实时预览翻译效果
- 批量编辑多语言内容
- 导入/导出翻译文件
- 团队协作编辑

## 配置说明

### 1. Intlayer 配置文件

项目根目录的 `intlayer.config.ts` 是 Intlayer 的核心配置文件：

```typescript
import { type IntlayerConfig, Locales } from "intlayer";

const config: IntlayerConfig = {
  internationalization: {
    locales: [Locales.CHINESE_SIMPLIFIED_CHINA, Locales.ENGLISH],
    defaultLocale: Locales.CHINESE_SIMPLIFIED_CHINA,
  },
};

export default config;
```

**配置项说明**：

- `locales`：支持的语言列表，使用 Intlayer 提供的 `Locales` 枚举
- `defaultLocale`：默认语言，用户首次访问时使用

### 2. Next.js 集成

在 `next.config.ts` 中使用 `withIntlayer` 包装配置：

```typescript
import { withIntlayer } from "next-intlayer/server";

const nextConfig = {
  // ... 其他配置
};

export default withIntlayer(nextConfig);
```

`withIntlayer` 会自动：
- 配置 Webpack/Turbopack 加载器
- 设置语言路由中间件
- 注入国际化上下文

### 3. 开发脚本配置

在 `package.json` 中配置开发脚本：

```json
{
  "scripts": {
    "dev": "intlayer watch --with 'next dev'",
    "dev:mock": "cross-env NEXT_PUBLIC_USE_MOCK=true intlayer watch --with 'next dev'",
    "start:editor": "intlayer-editor start --with 'next dev'",
    "editor": "intlayer-editor start"
  }
}
```

**脚本说明**：

- `intlayer watch`：监听 `.content.ts` 文件变化，自动重新生成类型定义
- `--with 'next dev'`：同时启动 Next.js 开发服务器
- `intlayer-editor start`：启动可视化编辑器

## 使用方式

### 1. 客户端组件

在客户端组件中使用 `useIntlayer` 钩子：

```typescript
"use client";

import { useIntlayer } from "next-intlayer";

export default function MyComponent() {
  const content = useIntlayer("my-component");
  
  return <div>{content.title}</div>;
}
```

### 2. 服务端组件

在服务端组件中使用 `useIntlayer` 的服务端版本：

```typescript
import { useIntlayer } from "next-intlayer/server";

export default async function MyServerComponent() {
  const content = await useIntlayer("my-component");
  
  return <div>{content.title}</div>;
}
```

### 3. 语言切换

Intlayer 提供了 `useLocale` 钩子用于语言切换：

```typescript
"use client";

import { useLocale } from "next-intlayer";
import { Locales } from "intlayer";

export default function LanguageSwitcher() {
  const { locale, setLocale } = useLocale();
  
  return (
    <select 
      value={locale} 
      onChange={(e) => setLocale(e.target.value as Locales)}
    >
      <option value={Locales.CHINESE_SIMPLIFIED_CHINA}>中文</option>
      <option value={Locales.ENGLISH}>English</option>
    </select>
  );
}
```

切换语言后，Intlayer 会：
- 更新 URL 路径（/zh → /en）
- 保存语言偏好到 Cookie
- 重新加载对应语言的翻译内容

### 4. 动态内容

对于需要插值的动态内容，可以使用函数形式：

```typescript
const content = {
  key: "greeting",
  content: {
    welcome: t({
      "zh-CN": (name: string) => `欢迎，${name}！`,
      en: (name: string) => `Welcome, ${name}!`,
    }),
  },
} satisfies DeclarationContent;
```

使用时：

```typescript
const { welcome } = useIntlayer("greeting");
return <div>{welcome("小明")}</div>;
```

### 字典文件组织

项目中的所有翻译内容都以 `.content.ts` 文件的形式组织，分布在各个模块中：

```
src/
├── app/
│   ├── [locale]/
│   │   ├── page.content.ts              # 首页翻译
│   │   ├── knowledge/
│   │   │   └── page.content.ts          # 知识库页面翻译
│   │   └── admin/
│   │       └── page.content.ts          # 管理后台翻译
│   └── layout.content.ts                # 布局翻译
├── components/
│   ├── Header.content.ts                # 头部组件翻译
│   ├── Footer.content.ts                # 底部组件翻译
│   └── KnowledgeCard.content.ts         # 知识卡片翻译
└── services/
    └── api.content.ts                   # API 错误消息翻译
```

### 生成的字典文件

Intlayer 会将所有 `.content.ts` 文件编译成 JSON 字典，存储在 `.intlayer/dictionary/` 目录：

```
.intlayer/
├── dictionary/
│   ├── home.json                        # 首页字典
│   ├── knowledge-list-page.json         # 知识库列表字典
│   ├── admin-dashboard.json             # 管理后台字典
│   └── ...
├── types/
│   ├── home.ts                          # 首页类型定义
│   ├── knowledge-list-page.ts           # 知识库列表类型定义
│   └── intlayer.d.ts                    # 全局类型定义
└── main/
    ├── dictionaries.mjs                 # 字典入口文件
    └── ...
```

## 最佳实践

### 1. 内容声明文件命名

- 使用 `.content.ts` 后缀
- 文件名与组件/页面名称保持一致
- 使用 kebab-case 命名（如 `knowledge-card.content.ts`）

### 2. 翻译键命名规范

- 使用语义化的键名（如 `title`、`description`、`submitButton`）
- 避免使用数字或无意义的键名（如 `text1`、`label2`）
- 嵌套对象表示层级关系（如 `nav.home`、`form.email.label`）

### 3. 内容组织原则

- 每个组件/页面独立管理自己的翻译内容
- 共享的翻译内容提取到公共文件（如 `common.content.ts`）
- 按功能模块组织，避免单个文件过大

### 4. 类型安全

- 始终使用 `satisfies DeclarationContent` 确保类型正确
- 利用 TypeScript 的类型推导，避免手动编写类型
- 在组件中使用 `useIntlayer` 时享受完整的类型提示

### 5. 性能优化

- 使用代码分割，按页面加载翻译内容
- 避免在循环中调用 `useIntlayer`
- 对于大型翻译内容，考虑懒加载

### 6. 翻译质量

- 保持中英文翻译的一致性和准确性
- 注意文化差异，避免直译
- 使用专业术语的标准翻译
- 定期审查和更新翻译内容

## 常见问题

### 1. 如何添加新语言？

在 `intlayer.config.ts` 中添加新语言：

```typescript
const config: IntlayerConfig = {
  internationalization: {
    locales: [
      Locales.CHINESE_SIMPLIFIED_CHINA,
      Locales.ENGLISH,
      Locales.JAPANESE,  // 添加日语
    ],
    defaultLocale: Locales.CHINESE_SIMPLIFIED_CHINA,
  },
};
```

然后在所有 `.content.ts` 文件中添加对应的翻译。

### 2. 如何处理复数形式？

Intlayer 支持复数形式的翻译：

```typescript
const content = {
  key: "items",
  content: {
    count: t({
      "zh-CN": {
        zero: "没有项目",
        one: "1 个项目",
        other: "{{count}} 个项目",
      },
      en: {
        zero: "No items",
        one: "1 item",
        other: "{{count}} items",
      },
    }),
  },
} satisfies DeclarationContent;
```

### 3. 如何处理日期和数字格式？

使用 JavaScript 的 `Intl` API：

```typescript
const formatDate = (date: Date, locale: string) => {
  return new Intl.DateTimeFormat(locale, {
    year: 'numeric',
    month: 'long',
    day: 'numeric',
  }).format(date);
};

const formatNumber = (num: number, locale: string) => {
  return new Intl.NumberFormat(locale).format(num);
};
```

### 4. 翻译内容不更新怎么办？

1. 确保 `intlayer watch` 正在运行
2. 检查 `.content.ts` 文件是否有语法错误
3. 重启开发服务器
4. 清除浏览器缓存

### 5. 如何在 API 响应中使用翻译？

在服务端使用 `getTranslation` 函数：

```typescript
import { getTranslation } from "next-intlayer/server";

export async function GET(request: Request) {
  const locale = request.headers.get("accept-language") || "zh-CN";
  const t = await getTranslation(locale, "api-messages");
  
  return Response.json({
    message: t.success,
  });
}
```

### 6. 静态导出时如何处理语言路由？

Next.js 的静态导出会为每种语言生成独立的 HTML 文件：

```
out/
├── zh/
│   ├── index.html
│   ├── knowledge/
│   │   └── index.html
│   └── ...
└── en/
    ├── index.html
    ├── knowledge/
    │   └── index.html
    └── ...
```

部署时，服务器会根据用户的语言偏好自动重定向到对应的语言目录。

### 7. 如何测试不同语言？

1. 手动切换语言：使用页面上的语言切换器
2. URL 参数：访问 `/zh` 或 `/en` 路径
3. 浏览器设置：修改浏览器的首选语言
4. Cookie：设置 `NEXT_LOCALE` Cookie

### 8. 可视化编辑器如何使用？

启动编辑器：

```bash
npm run start:editor
```

访问 `http://localhost:3000`（或开发服务器地址），编辑器会以侧边栏形式出现，可以：

- 点击页面上的文本进入编辑模式
- 在编辑器中修改翻译内容
- 实时预览修改效果
- 保存修改到 `.content.ts` 文件
