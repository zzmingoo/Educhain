## 项目简介

EduChain Web 是一个基于区块链存证的教育知识共享与智能检索系统的前端应用。采用 Next.js 16 + React 19 + Intlayer 构建，支持静态站点生成（SSG）和 Cloudflare Pages 部署。

### 项目定位

- **主要版本**: 当前活跃开发和维护的版本
- **技术方向**: 采用最新的 Next.js App Router 和 React Server Components
- **部署方式**: 静态站点生成，支持 CDN 加速
- **开发模式**: Mock 和真实后端双体系，支持独立开发

### 与旧版本的关系

本项目是 EduChain 前端的重构版本，相比旧版本（`apps/web-legacy`）有以下改进：

| 特性 | 旧版本 (web-legacy) | 新版本 (web) |
|------|-------------------|-------------|
| 框架 | React 18 + Vite | Next.js 16 + React 19 |
| 路由 | React Router v6 | Next.js App Router |
| 渲染 | CSR (客户端渲染) | SSG (静态站点生成) |
| 国际化 | react-i18next | Intlayer |
| Mock | 自定义 Mock | MSW (Mock Service Worker) |
| 部署 | 需要 Node.js 服务器 | 静态文件，支持 CDN |
| 开发时间 | 2025年3月-12月 | 2025年12月-2026年3月 |
| 状态 | 已停止维护 | 当前主要版本 |

## 核心特性

### 1. 双体系架构 🔄

**Mock 模式**
- 使用 MSW (Mock Service Worker) 拦截网络请求
- 完整的 Mock 数据和业务逻辑
- 无需后端即可独立开发和演示
- 支持网络延迟模拟

**真实后端模式**
- 通过环境变量一键切换
- 自动 Token 刷新机制
- 统一的错误处理
- 支持文件上传和下载

**切换方式**
```bash
# Mock 模式
NEXT_PUBLIC_USE_MOCK=true npm run dev

# 真实后端模式
npm run dev
```

### 2. 静态站点生成 📦

**优势**
- 构建时生成所有页面的 HTML
- 无需 Node.js 运行时
- 支持 CDN 加速
- 更好的 SEO 表现

**配置**
```typescript
// next.config.ts
export default {
  output: 'export',           // 启用静态导出
  images: {
    unoptimized: true,        // 禁用图片优化
  },
  trailingSlash: true,        // 添加尾部斜杠
}
```

### 3. 国际化支持 🌍

**支持语言**
- 中文简体（默认）
- 英文

**特性**
- 类型安全的翻译
- 自动生成 TypeScript 类型
- 支持动态切换
- 可视化编辑器

**使用示例**
```typescript
import { useIntlayer } from 'next-intlayer';

export default function Component() {
  const content = useIntlayer('component-name');
  return <h1>{content.title.value}</h1>;
```

## 项目背景

### 业务需求

EduChain 是一个教育知识共享平台，需要：

1. **知识管理**: 支持多种内容类型（文本、图片、视频、PDF、链接）
2. **区块链存证**: 为知识内容提供不可篡改的存证证明
3. **智能检索**: 全文搜索、分类筛选、标签匹配
4. **社区互动**: 评论、点赞、收藏、关注
5. **用户系统**: 个人资料、草稿管理、通知中心
6. **管理后台**: 用户管理、内容审核、系统设置

### 技术选型理由

**为什么选择 Next.js 16？**
- App Router 提供更好的路由体验
- Server Components 减少客户端 JavaScript
- 静态导出支持 CDN 部署
- 内置优化（图片、字体、代码分割）

**为什么选择 Intlayer？**
- 类型安全的国际化方案
- 与 Next.js 深度集成
- 支持 Server Components
- 可视化编辑器

**为什么使用 MSW？**
- 真实的网络请求拦截
- 与生产环境行为一致
- 支持浏览器和 Node.js
- 易于维护和测试

**为什么选择静态导出？**
- 降低服务器成本
- 提高访问速度
- 简化部署流程
- 更好的安全性
